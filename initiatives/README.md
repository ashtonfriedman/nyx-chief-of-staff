# Initiatives

Active efforts with goals, status, and next-actions. An initiative is anything with
a clear outcome and a timeline — features, projects, investigations, migrations.

## Template

```markdown
# {Initiative Name}

## Problem
(What needs to change and why)

## Approach
(How you plan to solve it)

## Status
(Current state — one line)

## Phases
(Break into milestones if multi-sprint)

## Open Questions
(What you don't know yet)
```

## Conventions

- One folder per initiative: `initiatives/{name}/{name}.md`
- Include `next-actions.md` for tasks with deadlines
- Link to ADO features/stories when they exist
- Move to `Archive/` when complete
- Status should be scannable in one line
